OpenAperture

Cloud Application Management Platform

OpenAperture is a free, open-source hybrid cloud management platform that delivers software quickly and consistently regardless of location or workload. This future-ready platform from Lexmark Enterprise Software provides a comprehensive management system to handle the six pillars of cloud management – provisioning, deployment, monitoring, maintenance, security and metering.


What are SystemEvents?

SystemEvents are notification events that are generated by all SystemComponents when something abnormal occurs in the system. SystemEvents may be retrieve with a GET to /system_events. Currently, events will be generated in the following situations:

  • An unhandled exception occurs during the processing of a message in any component
  • The Overseer is unable to determine how many nodes are in any cluster associated in the same MessagingExchange as the Overseer
  • The Overseer finds no available nodes are in any cluster associated in the same MessagingExchange as the Overseer
  • The Overseer finds a node in any cluster associated in the same MessagingExchange as the Overseer
    • has used more than 80% of the space available in the /var/lib/docker directory.
  • The Overseer finds a Fleet Unit instance on any node in any cluster associated in the same MessagingExchange as the Overseer
    • which is in a failed state.
  • The Overseer finds a Fleet Unit on any node in any cluster associated in the same MessagingExchange as the Overseer
    • which has no instances running.

SystemEvents may have one of the following severities:

  • error
  • warning
  • info
  • debug

SystemEvents may be dismissed and/or assigned any anyone registered in the OpenAperture system. An email will be sent to the assignee if the event is assigned to them, or previously was assigned and is being dismissed.
